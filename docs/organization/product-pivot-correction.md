# Orvo Product Pivot Correction

Date: 2026-05-24
Branch/worktree: `org/autonomous-org-research-v2` at `/root/orvo-agent-worktrees/autonomous-org-research-v2`

## Correction

The earlier autonomous research report in this branch was produced from an older prompt and still framed **Hito 0 / the 08:00 ARTEMEA WhatsApp report** as the priority trust wedge.

That framing is now superseded.

Orvo is being operated as a **sellable Atlassian-like ecommerce operations control plane**, not as a milestone/report-first project.

## Current strategic center

The product center is:

1. **Work Management Core** — `OperationalCase` / `WorkItem` as the native Jira-like object.
2. **Workflow Automation Platform** — deterministic triggers, conditions, actions, approvals, playbooks.
3. **Connector Platform** — registry-driven connectors with capabilities, health, scopes, contracts, emitted events/metrics.
4. **Semantic Intelligence** — metric/event/evidence definitions and deterministic insight-to-case mapping.
5. **Operator Surfaces** — queue, timeline, API, WhatsApp, digest/report as projections over cases/workflows.
6. **Trust/Admin/Security** — workspaces/businesses, RBAC, secret indirection, audit log, supportability.
7. **GTM/Packaging** — first sellable Tiendanube-first operations-control-plane SKU.

## Replacement priority stack

1. Define and implement `OperationalCase`/`WorkItem` lifecycle and contracts.
2. Define workflow automation primitives around cases.
3. Make connectors emit typed events/metrics with evidence lineage.
4. Wire semantic registry and deterministic insight-to-case mapping.
5. Build operator queue/timeline/API as the primary product surface.
6. Treat WhatsApp/reporting as projections over cases, not as source of truth.
7. Add tenant/admin/security/audit primitives so the product is sellable to teams.
8. Create GTM packaging and demo narrative around “operations control plane for Tiendanube-first brands.”

## Operating rule for all departments

If a worker has to choose between:

- making a report prettier, or
- strengthening cases/workflows/connectors/semantics/operator surfaces/trust/admin/GTM,

choose the product-platform primitive unless the report work is directly implemented as a projection of the case/workflow system.
